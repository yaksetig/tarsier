\subsection{Crate Organization}

\paragraph{Frontend.}
The \texttt{tarsier-dsl} crate implements a PEG parser (via the
\texttt{pest} library) that translates \texttt{.trs} source files
into an AST\@.  The \texttt{tarsier-ir} crate lowers the AST to
threshold automata, performing location expansion, shared-variable
allocation, rule generation, and property extraction
(Sect.~\ref{sec:lowering}).

\paragraph{Verification engine.}
The \texttt{tarsier-smt} crate provides a backend-agnostic SMT
interface with concrete backends for Z3~\cite{DeMoura2008} (via static
linking) and cvc5~\cite{BarbosaBBKLMMMN2022} (via a process-based
interface).  The \texttt{tarsier-engine} crate implements BMC,
$k$-induction, PDR, CEGAR~\cite{ClarkeGJLV2000}, and portfolio
solving.  It also provides partial-order reduction, symmetry reduction,
and counterexample trace extraction.

\paragraph{Probabilistic analysis.}
The \texttt{tarsier-prob} crate implements exact hypergeometric
probability computation using arbitrary-precision rational arithmetic.
For committee-based protocols such as
Algorand~\cite{GiladHMVZ2017, Micali2017}, it computes the maximum
adversary bound~$b_{\max}$ such that the committee corruption
probability remains below a target~$\varepsilon$, then injects this
bound into the SMT encoding.

\paragraph{Proof infrastructure.}
The \texttt{tarsier-proof-kernel} validates certificate bundles with a
minimal dependency footprint (Sect.~\ref{sec:certificates}).  The
\texttt{tarsier-certcheck} binary replays obligations against multiple
solvers.

\paragraph{Auxiliary crates.}
\texttt{tarsier-conformance} validates runtime execution traces
against the protocol specification without depending on the SMT
engine.  \texttt{tarsier-lsp} provides IDE integration via the
Language Server Protocol.  \texttt{tarsier-codegen} generates
protocol implementations from certified specifications.

\subsection{Solver Integration}

Z3 is accessed through thread-local contexts with Rust operator
overloading for arithmetic.  The incremental solving interface uses
push/pop scopes to extend the formula across $k$-induction depths.

For multi-solver portfolio mode, \tarsier{} spawns parallel solver
instances and takes the first definitive result.  The portfolio
strategy is particularly effective when Z3 and cvc5 have
complementary strengths on a given formula.

\subsection{CLI and Workflow}

The \texttt{tarsier-cli} provides commands for the full verification
workflow:
\begin{itemize}
  \item \texttt{verify}: BMC safety check up to depth~$k$.
  \item \texttt{prove}: Unbounded safety proof via $k$-induction or
    PDR\@.
  \item \texttt{prove-fair}: Liveness proof with weak/strong fairness.
  \item \texttt{analyze}: Multi-layer analysis with graduated
    confidence.
  \item \texttt{certify-safety}: Generate a proof certificate bundle.
  \item \texttt{export-ta}: Export the threshold automaton in ByMC
    format for cross-tool comparison.
\end{itemize}

Output is available in human-readable and JSON formats for CI
integration.
