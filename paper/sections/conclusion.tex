\section{Conclusion}\label{sec:conclusion}

We have presented \tarsier{}, a parameterized model checker for
threshold-guarded distributed protocols.  \tarsier{} advances the
state of the art in three directions: \emph{faithful network
semantics} via a hierarchy of four abstraction modes with formal
soundness guarantees; \emph{first-class cryptographic objects} with
SMT-encoded non-forgeability and signer-set constraints; and
\emph{proof-carrying verification} with a minimal, independently
validating proof kernel.

Our evaluation on 49~protocol models from 14~families shows that
\tarsier{} scales from minimal 4-location kernels ($<$50\,ms) to
complex 1024-location models ($<$2\,s), that faithful network modes
detect real bugs masked by classical counter abstraction, and that
proof certificates enable independent validation with a trusted
computing base of just four library dependencies.

Several directions remain open.  Extending the mode hierarchy to handle
\emph{dynamic committee membership} and reconfiguration would broaden
applicability to systems like Ethereum~2.0.  Integrating
\emph{fair-termination proofs} via ranking functions would strengthen
liveness guarantees beyond bounded lasso checking.  The
process-selective mode achieves instance-exact semantics but sacrifices
parameterized abstraction; developing \emph{bounded parametric}
techniques that combine process-selective precision with symbolic
generality is a promising direction.  Finally, connecting verified
specifications to \emph{certified code generation} would close the gap
between formal models and deployed implementations.
