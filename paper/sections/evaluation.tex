\section{Evaluation}\label{sec:evaluation}

We evaluate \tarsier{} on a corpus of 49~protocol models spanning
14~protocol families, addressing three research questions:
\begin{description}
  \item[RQ1] Does \tarsier{} scale to real-world BFT protocols?
  \item[RQ2] Do faithful network modes close soundness gaps present
    in classical abstractions?
  \item[RQ3] How do proof certificates affect the trusted computing
    base?
\end{description}

\subsection{Benchmark Corpus}

Table~\ref{tab:corpus} summarizes the corpus.  Models cover five major
protocol families---PBFT~\cite{CastroLiskov1999},
HotStuff~\cite{YinMGRGA2019}, Tendermint~\cite{BuchmanKM2018},
Paxos~\cite{Lamport1998}, and Algorand~\cite{GiladHMVZ2017}---plus nine
additional protocols (Streamlet, Casper~FFG, DLS, Zyzzyva, SBFT,
Narwhal/Bullshark, GRANDPA, QBFT, DiemBFT).  Each protocol is specified
in the \tarsier{} DSL with Byzantine, crash, or omission fault
semantics.  The corpus includes 14~\emph{regression sentinels}---models
with intentional bugs whose expected verdict is UNSAFE---to validate
bug-finding capability.

\subsection{Experimental Setup}

All experiments run on a single machine (Apple M3 Pro, 18\,GB RAM,
macOS~15) with Z3~v4.13 via static linking.  We report the median
wall-clock time over three runs.  Three benchmark profiles are used:
\begin{itemize}
  \item \textbf{Quick:} BMC depth~3, 30\,s timeout per layer.
  \item \textbf{Standard:} BMC depth~8, 120\,s timeout per layer.
  \item \textbf{Proof:} $k$-induction ($k {\leq} 16$) or PDR,
    180\,s timeout.
\end{itemize}

\subsection{RQ1: Scalability}

Table~\ref{tab:timing} reports verification times for representative
models.  \tarsier{} handles the full complexity spectrum: minimal
4-location models complete in under 50\,ms, while the 1024-location
Jolteon model finishes in 1.6\,s under quick mode.

The full 49-model corpus completes in under 3~minutes in quick mode.
All 25~models with expectation annotations pass: 11~expected-safe
protocols produce SAFE verdicts, and 14~regression sentinels correctly
expose violations.

The dominant cost factor is the location count~$|\mathcal{L}|$, which
determines the number of counter variables per step.  For the largest
model (Jolteon, 1024~locations), the SMT formula at depth~3 contains
${\sim}$12\,000 integer variables and ${\sim}$35\,000 assertions.
Z3 solves the resulting QF\_LIA instance in ${\sim}$1.5\,s per step,
confirming that counter abstraction keeps the problem tractable at
scale.

\subsection{RQ2: Faithful Network Modes}

We compare three modes on protocol variants that exist in both
classic and faithful (identity-selective or process-selective) forms.

\paragraph{Soundness gap detection.}
Five protocol families include paired safe/buggy variants under
faithful semantics (PBFT, HotStuff, Tendermint, Reliable Broadcast,
Viewstamped Replication).  In each case:
\begin{itemize}
  \item The classic-mode variant verifies SAFE.
  \item The faithful safe variant also verifies SAFE under
    identity-selective or process-selective mode.
  \item The faithful \emph{buggy} variant produces UNSAFE under
    identity-selective mode, detecting equivocation attacks that the
    classic mode cannot express.
\end{itemize}
This confirms Theorem~\ref{thm:procsel}: process-selective mode closes
the abstraction gap, enabling precise bug-finding where UNSAFE verdicts
correspond to real attacks.

\paragraph{Cryptographic object effectiveness.}
The crypto-object variants (PBFT Crypto~QC, HotStuff Crypto~QC,
Tendermint Crypto~QC) demonstrate the impact of first-class certificate
declarations.  With \texttt{conflicts exclusive}, the verifier
correctly rules out equivocation-based attacks that would be spuriously
flagged without certificate semantics.  Buggy variants that omit
conflict constraints correctly produce UNSAFE, confirming that the
encoding is both sound and discriminating.

\paragraph{Overhead of faithful modes.}
Identity-selective mode increases the shared-variable count
proportionally to the number of message types.  In practice, the
overhead is moderate: for PBFT with 3~message families,
identity-selective mode costs 1.3--1.8$\times$ classic mode.
Process-selective mode exhibits 2--4$\times$ overhead for small
instantiations ($n{=}4, t{=}1$) but does not scale to large parameter
domains since it gives up parameterized abstraction.

\subsection{RQ3: Proof Certificates}

We evaluate the proof-certificate infrastructure on the 11~expected-safe
models.

\paragraph{Certificate generation.}
All 11~models produce valid certificate bundles containing 2--3 SMT
obligations each.  Certificate generation adds $<$10\% overhead to
verification time.

\paragraph{Independent validation.}
The proof kernel validates all 11~bundles, checking schema
compatibility, hash integrity, and obligation completeness.  The
kernel comprises ${\sim}$800~lines of Rust with exactly four
dependencies.

\paragraph{Multi-solver replay.}
All obligations receive matching UNSAT verdicts from both Z3 and cvc5,
achieving the \emph{reinforced} assurance level
(Table~\ref{tab:assurance}).

\subsection{Cross-Tool Comparison}

\tarsier{} exports threshold automata in ByMC's \texttt{.ta}
format~\cite{KonnovWidder2018} and in Promela for
SPIN~\cite{Holzmann2003}.  We compared verdicts on five scenarios
(Reliable Broadcast safe/buggy, PBFT, Paxos, Tendermint):
\tarsier{} and the reference tools agree on all verdicts, confirming
model consistency.

\subsection{Threats to Validity}

\paragraph{Internal.}
Timing measurements depend on hardware and solver version.  We report
medians over three runs with deterministic replay scripts.

\paragraph{External.}
The corpus is curated by the tool authors.  Models are simplified
threshold-automata kernels, not full protocol implementations with view
changes or reconfiguration.  We partially mitigate this via variant
groups and regression sentinels.

\paragraph{Construct.}
Safety properties are expressed as conflicting location pairs.
Liveness is checked via bounded lasso analysis, which detects
violations only up to a bounded depth.  Unbounded fair-liveness
proofs are supported but not evaluated on all models.
